## 2024-05-20 - [CRITICAL] Fix Path Traversal Vulnerability in Tauri Filesystem Commands
**Vulnerability:** The Tauri filesystem commands (`fs_exists`, `fs_read_text`, `fs_write_text`, `fs_mkdir`) in `src-tauri/src/commands/fs_helpers.rs` allowed arbitrary paths to be used, which could lead to directory traversal if a user supplied `..` components in the path. Furthermore, the `fs_exists` command returned a boolean instead of `Result<bool, String>`, preventing proper security error handling across the application.
**Learning:** Security validation must be applied at the lowest possible level (in Rust commands) and errors must be explicitly returned to the frontend. Native Rust-side filesystem commands should perform their own security sanitization (`is_path_safe` helper blocking `..`) as they circumvent Tauri's plugin-fs scope rules.
**Prevention:** Implement a standard path validation helper like `is_path_safe` on all file-accessing Tauri commands, and make sure that any check operations, like `fs_exists`, return a `Result` so that validation errors can be properly reported back to the caller instead of just returning false or true blindly. Ensure the frontend correctly `.catch()`es the rejected Promise and handles it securely.
