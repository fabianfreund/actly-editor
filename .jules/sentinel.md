## 2026-03-29 - Missing Path Sanitization in Tauri FS Commands
**Vulnerability:** The Rust Tauri backend's filesystem commands (`fs_exists`, `fs_read_text`, `fs_write_text`, `fs_mkdir`, `fs_list_dir`, `open_in_vscode`) directly passed user-provided paths to `std::path::Path::new()` without verifying if they contained path traversal components like `..`.
**Learning:** This allowed potential directory traversal attacks, permitting arbitrary read/write/execution access beyond the intended project boundary if a malicious frontend sent paths like `../../../etc/passwd`. Traditional string-based path checking is insufficient; `Path::components()` provides an OS-agnostic, robust way to check for `ParentDir`.
**Prevention:** Always validate all user-supplied paths in native Tauri commands using `std::path::Component` parsing (e.g., checking for `Component::ParentDir`) before using them in `std::fs` operations, and ensure `fs_exists` returns `Result<bool, String>` to handle and report unsafe path errors consistently.
